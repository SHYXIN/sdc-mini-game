# ADR-0001: 技术栈选型

> 状态：已决定 | 日期：2026/05/19

## 背景

微信小游戏俯视角 2D 像素风撤离型射击游戏，初始包 4MB，分包总共 20MB，个人开发者。

## 决策

| 决策点 | 选型 | 理由 |
|--------|------|------|
| 引擎 | 纯 Canvas 自研 | 包体最小，无引擎依赖，2D 像素风不需要引擎的高级功能 |
| 语言 | TypeScript | 类型安全，IDE 提示好，适合 agent 协作 |
| 构建工具 | esbuild | 构建速度快，配置简单，输出单一 .js 文件适合小游戏 |
| 碰撞检测 | 圆形碰撞检测 | 俯视角射击游戏物体接近圆形，计算简单，性能足够 |
| 游戏循环 | 固定时间步长 | 保证不同帧率下逻辑一致，微信设备碎片化严重 |
| 输入系统 | 触摸 + 键盘抽象层 | PC 端调试方便，手机触摸和键盘输入统一接口 |
| 状态管理 | 状态栈（State Stack） | 支持层级状态（游戏中 → 暂停 → 弹窗），切换清晰 |
| 数据存储 | TypeScript 常量文件 | 零运行时加载，类型安全，编译时检查数据错误 |
| 音效 | Web Audio API 合成 | 零文件占用，像素风复古感，包体友好 |
| 渲染 | 分层 Layer + Y 轴排序 | 俯视角需要 Y 轴遮挡关系，分层保证大结构不乱 |

## 后果

- 包体控制在 4MB 以内可行
- 开发效率低于引擎方案，但灵活度最高
- 所有游戏系统需要从零实现，开发周期较长
- 后续如果包体有余量，可以补充轻量背景音乐 .mp3
